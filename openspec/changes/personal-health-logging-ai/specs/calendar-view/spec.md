## ADDED Requirements

### Requirement: 달력에서 건강 기록이 있는 날짜를 표시한다
달력 화면 SHALL health_record가 존재하는 날짜에만 마킹을 표시해야 한다. 기록이 없는 날은 마킹하지 않는다.

#### Scenario: 기록 있는 날짜 마킹
- **WHEN** 특정 날짜에 health_record가 1개 이상 존재한다
- **THEN** 달력에서 해당 날짜에 시각적 마킹(점 또는 아이콘)이 표시된다

#### Scenario: 기록 없는 날짜 마킹 없음
- **WHEN** 특정 날짜에 health_record가 존재하지 않는다
- **THEN** 해당 날짜에 아무 마킹도 표시되지 않는다

### Requirement: 날짜 클릭 시 해당 날의 기록 이력을 표시한다
사용자가 달력에서 날짜를 클릭하면 해당 날짜에 등록된 모든 health_record를 목록으로 표시해야 한다.

#### Scenario: 날짜 클릭 시 이력 표시
- **WHEN** 사용자가 기록이 있는 날짜를 클릭한다
- **THEN** 해당 날짜의 health_record 목록이 각각 부위 아이콘, 증상 텍스트, 발생 시각, 컨텍스트와 함께 표시된다

#### Scenario: 기록 없는 날짜 클릭
- **WHEN** 사용자가 기록이 없는 날짜를 클릭한다
- **THEN** "이 날은 기록된 증상이 없습니다" 메시지와 함께 새 기록 추가 버튼이 표시된다

### Requirement: 달력 상세에서 기록을 수정하거나 삭제할 수 있다
사용자 SHALL 달력 상세 화면에서 각 health_record를 수정하거나 삭제할 수 있어야 한다.

#### Scenario: 기록 수정
- **WHEN** 사용자가 특정 기록의 수정 버튼을 누른다
- **THEN** 편집 폼이 나타나고 부위, 증상, 시간, 컨텍스트를 수정 후 저장할 수 있다

#### Scenario: 기록 삭제
- **WHEN** 사용자가 특정 기록의 삭제 버튼을 누른다
- **THEN** 확인 다이얼로그 후 해당 health_record가 삭제된다

### Requirement: 달력 진입 시 오늘 날짜 세부 내역을 자동으로 표시한다
달력 화면 진입 시 URL param이 없으면 오늘 날짜의 세부 이력 패널이 자동으로 열려야 한다.

#### Scenario: 달력 탭 진입 시 오늘 날짜 자동 오픈
- **WHEN** 사용자가 URL param 없이 달력 화면에 진입한다
- **THEN** 오늘 날짜가 선택된 상태로 해당 날의 기록 패널이 자동으로 표시된다

#### Scenario: 채팅에서 "달력에서 확인하기" 버튼으로 진입 시 해당 날짜 오픈
- **WHEN** 사용자가 채팅의 "달력에서 확인하기" 버튼을 눌러 달력으로 이동한다
- **THEN** 해당 기록의 날짜 패널이 즉시 열리고 기록 목록이 표시된다

### Requirement: 달력은 월 단위로 탐색 가능하다
사용자 SHALL 달력에서 이전 달·다음 달로 이동하며 기록을 확인할 수 있어야 한다.

#### Scenario: 이전 달 이동
- **WHEN** 사용자가 달력에서 이전 달 버튼을 누른다
- **THEN** 이전 달의 달력이 표시되고 해당 월의 기록 마킹이 반영된다
