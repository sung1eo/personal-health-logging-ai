## ADDED Requirements

### Requirement: 채팅이 앱 첫 화면으로 제공된다
앱 진입 시 채팅 UI가 첫 화면으로 표시되어야 한다. 사용자는 텍스트 또는 음성으로 입력할 수 있어야 한다.

#### Scenario: 앱 첫 진입 시 채팅 화면 표시
- **WHEN** 사용자가 앱에 접속한다
- **THEN** 채팅 입력창과 대화 히스토리 영역이 바로 표시된다

#### Scenario: 텍스트 입력
- **WHEN** 사용자가 텍스트 입력창에 메시지를 입력하고 전송한다
- **THEN** 메시지가 대화 히스토리에 표시되고 AI 응답이 반환된다

#### Scenario: 음성 입력
- **WHEN** 사용자가 마이크 버튼을 눌러 음성으로 메시지를 입력한다
- **THEN** 음성이 텍스트로 변환되어 전송되고 AI 응답이 반환된다

#### Scenario: 음성 미지원 환경
- **WHEN** 브라우저가 Web Speech API를 지원하지 않는 환경에서 접속한다
- **THEN** 마이크 버튼이 비활성화되고 텍스트 입력만 제공된다

### Requirement: 기록 완료 후 추가 컨텍스트 유도 질문을 한다
AI SHALL 증상을 기록한 직후 시간 및 상황에 대한 추가 질문을 통해 기록을 보완해야 한다.
사용자가 시간/상황을 추가 제공하면 `update_health_record`로 기록을 업데이트한다.

#### Scenario: 기록 후 추가 질문
- **WHEN** AI가 증상을 health_record로 저장한다
- **THEN** "몇 시쯤, 어떤 상황에서 그러셨는지 알려주시면 도움이 돼요" 형태의 추가 질문을 함께 제공한다

#### Scenario: 추가 정보 제공 시 기록 업데이트
- **WHEN** 사용자가 "아까 오후 2시쯤이었어요"처럼 추가 정보를 제공한다
- **THEN** AI가 해당 health_record의 occurred_at 또는 context_text를 업데이트한다

### Requirement: 기록 완료 후 달력 바로가기 버튼 제공
AI가 증상을 성공적으로 기록한 후 채팅 응답 안에 "달력에서 확인하기" 버튼을 제공해야 한다.

#### Scenario: 증상 기록 완료 시 버튼 노출
- **WHEN** AI가 사용자 증상을 health_record로 저장 완료한다
- **THEN** AI 응답 메시지 하단에 해당 날짜 달력 상세 화면으로 이동하는 버튼이 표시된다

#### Scenario: 버튼 클릭 시 달력 이동
- **WHEN** 사용자가 "달력에서 확인하기" 버튼을 클릭한다
- **THEN** 해당 기록의 날짜 달력 상세 화면으로 이동한다

### Requirement: 대화 히스토리 유지
같은 세션 내에서 이전 대화 내용이 유지되어야 하며 AI는 이전 대화 컨텍스트를 참고하여 응답해야 한다.
화면 이동(달력, 마이 등) 후 채팅으로 돌아와도 대화 이력과 conversationId가 복원되어야 한다.
대화 이력과 conversationId는 localStorage에 저장하여 페이지 리마운트 시 복원한다.

#### Scenario: 세션 내 컨텍스트 유지
- **WHEN** 사용자가 대화 중 "아까 말한 증상"처럼 이전 내용을 참조한다
- **THEN** AI가 이전 대화 내용을 기반으로 적절히 응답한다

#### Scenario: 화면 이동 후 채팅 복귀 시 이력 유지
- **WHEN** 사용자가 달력 화면으로 이동했다가 채팅 탭을 다시 누른다
- **THEN** 이전 대화 메시지 목록이 그대로 표시되고 같은 conversation이 이어진다
